# 🧠 Memory Bridge

Built for the Microsoft x Women in Cloud AI Innovation Challenge

We didn't build for the average case. We built for the hardest one. Because if you can reduce cognitive load for someone with dementia, you can reduce it for anyone.

---

## The Problem

Over 55 million people worldwide live with dementia. The tools that exist treat it as a medical and logistics problem — medication reminders, GPS trackers, appointment schedulers. But nobody is solving the human problem.

As cognitive decline progresses, families don't just lose their loved ones to memory loss. They lose the shared language, stories, and references that made their relationship unique. A mother forgets her daughter's name but still lights up when she hears Michael Jackson. A father can't remember what year it is but tells the same story about his first car with perfect detail every time.

Caregivers grieve in real time while still providing care, with almost no tools designed for the emotional side of this experience.

Most tools treat dementia as a storage problem. The real problem is a relationship problem.

---

## Our Approach

Memory Bridge was built in response to the Cognitive Load Reduction challenge. We believe dementia represents the most extreme form of cognitive overload that exists.

It is not just information overload. It is cognitive loss: the gradual erosion of the very tools people use to process the world. Every principle in the brief applies here in its most urgent form: simplified language, adaptive communication, calm and dignity-preserving interactions, personalized memory anchors, and support that meets people exactly where they are.

## Features

### 🟢 Memory Harvesting Interview

Before significant cognitive decline, the system conducts gentle, conversational interviews to capture life stories, personal preferences, family connections, favourite places, and meaningful moments.

- Powered by Azure OpenAI for natural, empathetic conversation
- Stores memories in Azure Cosmos DB for real-time retrieval
- As the interview progresses, the system summarizes what it has learned in structured bullet points — simplifying complex personal history into clear, digestible preferences. This is the cognitive load reduction principle in action: taking unstructured conversation and returning it as organized, actionable insight.

### 🤖 Adaptive Daily Companion

The AI engages the patient daily using their own memories as emotional anchors.

- Dynamically adjusts language complexity based on the patient's state
- Three modes: Gentle, Normal, and Direct, switchable in real time
- Never corrects, never alarms, never makes the patient feel lost
- Azure AI Content Safety ensures all interactions remain calm and non-distressing
- Meets them exactly where they are, not where they used to be

### 📊 Caregiver Emotional Intelligence Layer

The system supports caregivers through a dynamic dashboard that surfaces real insights from conversations.

- Weekly mood summaries generated by AI from actual conversation data
- Patient name, mood score, and insights are all dynamic — not hardcoded
- Azure Monitor and Application Insights track interaction patterns over time
- Azure Logic Apps triggers event-driven workflows such as caregiver notifications after meaningful interactions
- Caregivers receive a warm, human-centered update rather than a clinical report

### 🔒 Consent and Data Sovereignty

Because this data is deeply personal, patients remain in full control.

- Toggle sharing with caregivers on or off
- Toggle AI learning on or off
- Delete all data permanently with a single confirmed action
- Credentials secured using Azure Key Vault with role-based access control
- Principle of Least Privilege applied — each component only has the minimum permissions required

## Azure Architecture

| Service | Role |
|---|---|
| Azure OpenAI | Adaptive conversation and memory harvesting |
| Azure Cosmos DB | Stores the evolving personal memory model |
| Azure AI Content Safety | Ensures all language is calm and non-distressing |
| Azure App Service | Hosts the deployed backend |
| Azure Key Vault | Secures credentials with role-based access control |
| Azure Monitor | Tracks interaction patterns and cognitive signals |
| Azure Logic Apps | Caregiver nudges, notifications, and weekly digests |
| Azure Translator | Accessibility — translates responses across languages |

Azure Logic Apps, Key Vault, and Azure Monitor are configured and active in our production deployment on Azure App Service. These are infrastructure-level services that do not surface visually in the demo, but they are running: Key Vault securing credentials, Monitor logging every interaction, and Logic Apps triggering caregiver notifications after meaningful sessions.

## Running Locally (Development)

### Prerequisites
- Node.js 18+
- Azure OpenAI credentials

### Steps
```bash
# 1. Clone the repo
git clone https://github.com/n0va-ctrl/memory-bridge.git
cd memory-bridge

# 2. Install dependencies
npm install

# 3. Add your environment variables
# Create a .env.local file in the memory-bridge folder with the variables listed below

# 4. Start the development server
npm run dev -- --webpack

# 5. Open in browser
http://localhost:3000/onboarding
```

> Note: Use `npm run dev -- --webpack` instead of `npm run dev` if you are on Apple Silicon (M1/M2/M3 Mac)

### Reset memories
```bash
echo "[]" > data/patientMemory.json
```
